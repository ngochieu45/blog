Understanding Async/Await in Depth

Alex Johnson · April 10, 2026 · 5 min read

Async/await is one of the most powerful features in modern JavaScript, but it's often misunderstood. In this post, we'll go beyond the basics and look at what's really happening under the hood.

What is async/await?

At its core, async/await is syntactic sugar over Promises. When you mark a function as async, it automatically returns a Promise. The await keyword pauses execution inside that function until the Promise resolves.

async function fetchUser(id) {
  const response = await fetch(`/api/users/${id}`);
  const user = await response.json();
  return user;
}

Common Pitfalls

One of the most common mistakes is using await inside a forEach loop. This doesn't work as expected because forEach doesn't await the callback.

// ❌ This won't work as expected
items.forEach(async (item) => {
  await processItem(item);
});

// ✅ Use for...of instead
for (const item of items) {
  await processItem(item);
}

Parallel Execution

When you have independent async operations, run them in parallel with Promise.all instead of awaiting them sequentially.

// ❌ Sequential – slow
const user = await getUser(id);
const posts = await getPosts(id);

// ✅ Parallel – fast
const [user, posts] = await Promise.all([getUser(id), getPosts(id)]);

Understanding these patterns will make your async code both more correct and more performant. Happy coding!
